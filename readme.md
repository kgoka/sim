## 🚧 1. 협업 가이드라인

### 1️⃣ 워크플로우 (Workflow)
* **Gitflow** 방식을 사용합니다. (모르면 '코딩애플' 참고!)
* `main`이나 `develop` 브랜치에 **직접 Push 절대 금지** ❌ (참사 내지 마세요..)
* 작업 시 `develop` 브랜치에서 따와서 **`feature/기능명`** 브랜치를 생성하여 작업합니다.

### 2️⃣ 커밋 메시지 규칙 (Commit Message)
커밋할 때 "수정", "추가" 처럼 대충 적지 말고 아래 말머리 규칙을 지겨주세요.

| 말머리 | 설명 | 예시 |
| :--- | :--- | :--- |
| **`feat`** | 새로운 기능 추가 | `feat: 농구선수 결과 페이지 제작` |
| **`fix`** | 버그 수정 | `fix: 점수 계산 오류 수정` |
| **`design`** | CSS 등 사용자 UI 디자인 수정 | `design: 메인 버튼 색상 변경` |
| **`refactor`** | 기능 변경 없는 코드 구조 개선 | `refactor: 변수명 정리` |

### 3️⃣ 풀 리퀘스트 (Pull Request)
* 작업이 끝나면 `feature` → `develop` 브랜치로 **Pull Request(PR)**를 날립니다.
* **PR 없이 바로 Merge 금지!** 🚫
* 서로 코드 리뷰(Comment) 남기고 승인되면 머지합니다.
